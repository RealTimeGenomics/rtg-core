Classes associated with the SLIM logging, error, warning, and progress reporting systems.

The main class controlling the diagnostic system is Diagnostic. It contains static methods for registering with the diagnostic system and methods to generate various kinds of diagnostics.

How to add a new diagnostic

First determine whether your diagnostic is an error, warning, or progress event. Create an extra value in the appropriate enumeration; that is, ErrorType, WarningType, or ProgressType, respectively. The new entry should specify the number of parameters associated with the warning message. Update the corresponding test class.

Next, add a default message to resource bundle (Diagnostics.properties), using %1, %2, etc., to indicate positions where the parameters are to be substituted into the message. Add foreign language translations as required.

Run some tests to make sure your message is working as expected. This includes the tests in ErrorTypeTest

Run the checkstyle-extras-jar ant task in the build.xml file. This creates a new internal/checkstyle-cartesian.jar file (in Eclipse you will need to refresh this). Copy this to your Eclipse applications directory (something like eclipse3.2/plugins/com.atlassw.tools.eclipse.checkstyle_4.3.2/extension-libraries). Exit from Eclipse and do a clean start.
